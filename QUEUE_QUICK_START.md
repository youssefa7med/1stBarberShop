## الخوارزمية البسيطة

### كيفية عمل الحساب؟

```
الخطوة 1: جلب جميع حجوزات اليوم (pending + ongoing)
├─ ترتيبها من الأقدم للأحدث

الخطوة 2: فصل الحجوزات
├─ الحجز الحالي (ongoing)
├─ الحجوزات المنتظرة (pending)

الخطوة 3: الحساب
├─ إذا كان هناك حجز جاري:
│  └─ وقت الانتظار = الوقت المتبقي له
├─ أضف مدة جميع الحجوزات المنتظرة
└─ أضف الكل للوقت الحالي = الوقت المتوقع
```

### مثال عملي

```
الوقت الآن: 10:00 ص

الحجوزات:
1. أحمد (جاري)
   - الوقت: 9:30 ص
   - المدة: 30 دقيقة
   - ينتهي في: 10:00 ص
   - الوقت المتبقي: 0 دقائق ✅ انتهى

2. محمد (انتظار)
   - الوقت: 10:00 ص
   - المدة: 30 دقيقة

3. علي (انتظار)
   - الوقت: 10:30 ص
   - المدة: 20 دقيقة

الحساب:
- أمامك: 2 شخص
- الانتظار: 30 + 20 = 50 دقيقة
- الوقت المتوقع: 10:00 + 50 = 10:50 ص
```

## نصائح الأداء

### ✅ يجب القيام به:
- ✔️ حدّث حالة الحجز عندما تبدأ/تنتهي الجلسة
- ✔️ أغلق الصفحة عند عدم الاستخدام
- ✔️ استخدم HTTPS في الإنتاج
- ✔️ اختبر على أجهزة حقيقية

### ❌ تجنب:
- ❌ لا تحدث الصفحة بسرعة فائقة (<500ms)
- ❌ لا تترك النافذة مفتوحة بدون استخدام
- ❌ لا تضغط على F5 بشكل متكرر

## استكشاف المشاكل الشهيرة

### المشكلة: الأرقام لا تتغير
**الحل:**
```
1. تأكد من إضافة حجز جديد فعلاً
2. تحقق من حالة الحجز (pending أو ongoing)
3. أعد تحميل الصفحة F5
4. تحقق من الكونسول (F12) لا توجد أخطاء
```

### المشكلة: الوقت غير صحيح
**الحل:**
```
1. تحقق من توقيت النظام
2. شغّل getEgyptDateString() في الكونسول
3. تأكد من timezone صحيحة (EGY)
4. أعد تشغيل الخادم
```

### المشكلة: الألوان غير الصحيحة
**الحل:**
```
1. امسح ذاكرة التخزين (Ctrl+Shift+Delete)
2. أعد تحميل الصفحة والتطبيق
3. تحقق من ملفات CSS
4. أعد بناء المشروع (npm run build)
```

## أمثلة الاستخدام

### مثال 1: عرض على شاشة الاستقبال
```
1. افتح متصفح على كمبيوتر الاستقبال
2. ادخل الرابط: http://localhost:5173/queue

```

```
3. أضف/عدّل الحجوزات
4. الطابور يحدّث نفسه تلقائياً
```
